## 1. 结论摘要

- 当前 A股字段级有效覆盖率：**9%**。
- 当前港股字段级有效覆盖率：**9%**。
- 当前 A/H 数据只能支持低置信度的价格方向观察，不能支持高置信度估值、基本面、市场宽度、资金流或精确金额建议。
- A/H 行情实际成功源均为 **yfinance 单源**；Finnhub 是代码中的优先源，但当前快照未成功形成第二来源。
- `03033` 的映射存在 P0 错误：真实持仓是南方东英恒生科技指数 ETF（HKEX 03033/行情代码通常为 `3033.HK`），当前 Opportunity Score 却使用 `3067.HK`。HKEX 资料显示 `3067` 是另一只 iShares Hang Seng TECH ETF，不能当作同一证券。
- `513060.SS`、`513090.SS`、`002558.SZ` 未进入行情路由；对应评分因行情、估值、基本面和资金流缺失而不可信，只能保留“观察”。
- 恒生指数、恒生科技指数、恒生医疗保健指数和沪深300指数本身均未接入。当前报告使用 ETF 代理，不能冒充指数行情。
- A/H 的 `market_timezone` 被统一写成 `America/New_York`，与时间戳中的 `+08:00` 冲突。最新快照的市场日期为 2026-07-13，数值可作为上一交易日收盘参考，但时区元数据不合格。

## 2. 实际生产调用链

```text
main.py
  -> src.app.main()
  -> src.app.run()
  -> scripts.build_daily_snapshot.write_snapshot()
  -> utils.market_data_provider.fetch_yfinance_market_data()
  -> src.data_sources.data_router.fetch_layered_market_data()
     -> Finnhub / yfinance（A股、港股）
     -> 本地 JSON 缓存（全部在线源失败时）
  -> src.data_sources.source_audit.build_and_write_source_audit()
  -> src.decision.v12_1_decision.build_v12_1_decision()
     -> Risk Score
     -> Opportunity Score
  -> src.reports.report_center.generate_daily_report()
```

GitHub Actions 实际运行 `python main.py`，与本地正式入口一致。用户要求中列出的 `src/data_fetchers/` 与 `src/scoring/` 在仓库中不存在；实际对应目录是 `src/data_sources/`，评分逻辑位于 `src/decision/v12_1_decision.py`。

## 8. 仅展示但未真正进入决策的数据

- `3033.HK` 和 `2800.HK` 被抓取，并可进入跨资产摘要，但不进入对应真实持仓的 Opportunity Score。
- `market_data.csv` 中“沪深300涨跌幅”“恒生科技涨跌幅”仍是手工/旧兼容输入，值为 0.00；它们参与 legacy MarketAgent 和回退文字，不应被理解为实时市场值。
- 来源白名单中的 HKEX、SSE、SZSE 只完成登记，没有实际请求或数据返回。
- Finnhub 的“优先源”地位存在于代码，但失败原因没有完整保留到 `source_audit.json`；审计文件可能显示“failed_sources=[]”，不能据此断言 Finnhub 成功。
- 资金流、市场宽度、AAII、Put/Call 在报告中仅显示“未接入”，不会形成有效分值；Opportunity 的“资金流”实际使用成交量比率，不能等同 ETF 净流入。

## 9. 因数据不足而不可信的评分

- **巨人网络、恒生医疗ETF、香港证券ETF**：行情缺失，最终分主要由类型默认值、组合偏离和惩罚构成。
- **恒生科技ETF**：使用错误证券 `3067.HK`，即使价格有效，评分事实基础也不正确。
