## 数据源选择链路

1. Tushare Pro：接口可用且有权限时优先。
2. 官方源：HKMA、巨潮资讯、HKEX及中证指数等正式核验源。
3. AKShare：受监控备用源，必须披露底层提供方并通过结构、日期、单位、时效与冲突校验。
4. yfinance：保留既有基础行情职责，并可参与同交易日价格核验。
5. 本地缓存：仅在接口失败后使用；过期缓存不得支持高置信度建议。

## 新增稳定性保护

- 每个接口硬超时，最多重试2次。
- 每个接口连续失败3次后本轮熔断，不连带误伤其他接口。
- 空响应、字段结构变化、网络错误、SSL错误和超时独立分类。
- 成功数据写入JSON缓存；缓存记录年龄、是否过期和上游失败原因。
- 缺失值保留为`null`，不填0。
- 主源与备用源仅在同交易日、同币种且主源未过期时比较。
- 价格差异超过5%或估值差异超过20%标记`SOURCE_CONFLICT`，相关记录禁入评分并降低DQS。
- AKShare记录必须包含`source=akshare`及`underlying_provider`，不把聚合器描述成官方源。

## 002558字段处理

估值和财务分开存储。当前已记录PE、PB、总市值、营业收入、归母净利润、收入同比、净利润同比、ROE、经营现金流每股、资产负债率和报告期。经营现金流总额仍缺失。不同报告期数据不会合并；报告期不一致时`period_consistent=false`且禁止进入评分。

当前002558估值响应缺少明确市场日期，因此PE/PB/总市值只展示。财务摘要报告期明确，进入基本面评分的字段为ROE、净利率、资产负债率、收入同比和净利润同比。营业收入、归母净利润和每股经营现金流保留在快照中供展示与审计，不替代缺失的经营现金流总额。
