# Stone AI Investment Manager Pro V12 Agent Rules

本文件是项目内长期规则，不改变 Stone AI 稳定版名称、版本号、投资目标和目标配置。

## 职责边界

- Codex 负责：数据抓取、清洗、资产台账、执行状态、规则校验、预检、快照生成和候选建议。
- GPT 负责：CIO 复核、风险判断、冲突仲裁、例外机制解释和最终用户决策。
- Codex 不得取代 GPT 做最终投资裁决。
- 系统不自动交易，不接券商下单权限，不承诺收益。

## 五个业务任务边界

- 每日CIO：读取统一快照并生成候选建议；非周三且无特殊机会时默认不交易。
- 周三定投：唯一基础定投任务；同一自然周最多一次基础定投。
- 风险委员会：只在重大风险、blocking_errors 或 DQS 严重不足时通知。
- 全球监控：只处理独立高价值机会；不得把普通波动包装成交易信号。
- 周度复盘：核对建议、待确认、已执行和已取消状态；未确认交易不得入账。

## 数据源优先级

1. `data/portfolio_master.yaml`
2. 券商和成交记录
3. `data/execution_state.json`
4. 市场数据
5. Codex 候选建议
6. Gmail 邮件
7. 新闻解释

Gmail 只能作为报告和通知渠道，不得作为资产事实或成交事实的唯一来源。

## DQS 硬门槛

- DQS >= 85：允许输出精确金额。
- DQS 70-84：只能输出方向、比例或金额上限。
- DQS < 70：不得给交易建议。
- `blocking_errors` 非空：停止执行单，只输出阻断原因和需要修复的数据项。

## 失败降级

- IBKR 未接通：成交事实不能自动入账，只能等待用户确认或券商记录。
- 市场数据缺失：降级到缓存或手动数据；若 DQS 不达标，不给交易建议。
- 金价失败：保留主台账或手动估值，并标记估值状态；不得重复计算黄金。
- 债券明细失败：10年地债已包含在中国债券中，严禁重复加总。
- 邮件失败：不影响报告生成，但必须在日志和系统检查报告中提示。

## 执行状态

`data/execution_state.json` 必须区分：

- `suggested`：建议
- `pending_confirmation`：待确认
- `executed`：已执行
- `cancelled`：已取消

订单类型必须区分：

- `base_dca`：基础定投
- `opportunity_add`：机会加仓

未确认交易不得写入资产台账。基础定投同一自然周最多一次。

## 全球权威数据扫描与来源审计

- 来源白名单固定在 `config/source_registry.yaml`，所有宏观、市场、指数、公司公告、资金流和新闻来源必须先登记后使用。
- 官方原始来源优先于媒体和聚合平台；当官方源与媒体源冲突时，采用官方源，并在 `data/source_audit.json` 记录冲突和取舍。
- 未登记来源不得用于形成交易结论；可以出现在日志中，但不能进入执行单依据。
- 价格、VIX、收益率、汇率等关键数据必须双源验证；只有单一非官方来源时，不允许给精确金额。
- 每次运行必须生成 `data/source_audit.json`，记录计划来源、成功来源、失败来源、过期来源、一级来源覆盖率、关键指标覆盖率、数据冲突和响应摘要。
- 一级来源覆盖率低于 80%，或关键指标覆盖率低于 85% 时：输出“数据覆盖不足”，DQS 不得高于 69，禁止精确金额建议。
- 每个重要数字必须带来源与时间戳；无法确认时间戳时视为可能过期。
- 交易候选必须满足三类独立证据：市场/估值、宏观/基本面、资金流/波动率/行为。缺少任一类时，只能输出观察或待复核。
- Gmail 仍只作为报告和通知渠道，不得作为资产事实、成交事实或行情事实的唯一来源。
