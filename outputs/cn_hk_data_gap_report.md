# A股与港股数据缺口报告

审计日期：2026-07-14  
适用版本：Stone AI Investment Manager V12.6 Stable

## 当前结论

| 市场 | 字段级有效覆盖率 | 用户持仓直接行情覆盖 | 评分可信度 |
|---|---:|---:|---|
| A股 | 9% | 1/2 | 沪深300ETF可作单源方向参考；巨人网络不可信 |
| 港股及港股主题基金 | 9% | 0/3 | 03033映射错误；513060、513090行情缺失 |

当前 A/H 数据不足以支持高置信度买入建议。系统现有 DQS 是全球混合口径，不能替代 A股和港股各自的数据完整度门槛。

## P0：必须先修复

### 1. 03033 行情代理错误

- 用户真实持仓：南方东英恒生科技指数 ETF，代码 `03033`。
- HKEX 对应交易代码：`3033`；yfinance 通常使用 `3033.HK`。
- 当前评分代码：使用 `3067.HK`。
- `3067` 是另一只 iShares Hang Seng TECH ETF，不是同一证券。
- 影响：恒生科技ETF的价格、成交量、趋势、资金结构和 Opportunity Score 均引用了错误证券。

处理原则：修正为“真实持仓 `03033` / 直接行情 `3033.HK`”；如需指数代理，应另设 `HSTECH_INDEX_PROXY`，不得混为持仓价格。

### 2. 三个真实持仓没有行情请求

`MARKET_TICKERS` 与 `HK_CN_TICKERS` 缺少：

- `002558.SZ` 巨人网络
- `513060.SS` 恒生医疗ETF
- `513090.SS` 香港证券ETF

这些代码已存在于 Security Master 和 Opportunity Score，但数据路由不抓取，因此分数主要来自默认模板和惩罚项。

### 3. A/H 时区元数据错误

所有 yfinance 数据被按“来源”而不是“交易所”标记为 `America/New_York`。A股和港股应使用 `Asia/Shanghai` / `Asia/Hong_Kong`。错误时区会影响交易日判断、过期判断和“最新交易日”判断。

### 4. A/H 专项数据门槛缺失

当前 Source Audit 的关键指标只有 `3067.HK` 和 `510300.SS` 两项 A/H 代理。只要这两项有值，全局关键数据覆盖率就可能较高；这不能代表用户五个 A/H 主要持仓和四个指数的完整覆盖。

建议增加只读质量输出：

- `cn_market_coverage_pct`
- `hk_market_coverage_pct`
- `cn_critical_missing`
- `hk_critical_missing`
- `cn_hk_trade_confidence_gate`

任何市场的关键持仓行情覆盖不足时，该市场候选只能“观察/等待数据”，不能给高置信度金额建议。

## P1：最小稳定数据接入

1. 接入 Tushare Pro 适配器，优先覆盖 A股日线、复权因子、每日指标、财务报表、财报日历、指数行情与估值。
2. yfinance 保留为报价便利源和第二来源，不作为 A/H 唯一高置信度事实源。
3. AKShare 仅作为备用源，所有接口单独做超时、字段校验和回归测试，不将网页结构依赖接口提升为一级来源。
4. 接入 HKMA 官方公开 API：HIBOR、总结余、港元流动性。
5. 接入 HKEX/SSE/SZSE/巨潮的公告与日度统计，只使用明确、稳定、许可允许的公开接口或文件。
6. 增加沪深300、恒生指数、恒生科技指数、恒生医疗保健指数的独立 canonical_id，严格区分指数、ETF和行情代理。

## P2：增强而非前置依赖

- A股上涨/下跌家数、涨跌停、行业强弱
- 港股市场宽度、卖空比例、南向成交统计
- ETF份额/AUM变化和可靠资金流
- AH股溢价
- 盈利预测和一致预期

以上数据可提高解释力，但不应为了“凑覆盖率”引入来源不明的网页抓取。
